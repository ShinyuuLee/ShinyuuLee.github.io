Tricks for CNN Coding

Nov 13, 2017

After knowing the basic architectures of some classic CNNs, I think itâ€™s helpful to master several tips or tricks when we are implementing these models on computers. These tricks could help us to reduce running time, improve operational efficiency and even optimize model stucture. In this way, we also could adjust some parameters such as learning rate much faster to get better consequences in shorter time. Here we continue to use Gluon of Python as a tool to make it.

What we use includes and not only includes:

  • Parameter Delay Initialization;
  • Batch Norm;
  • Dropout;
  • Hybridize;
  • Gpu Acceleration;
  • Image Augmentation;
  • Weight-Decay Regularization;
  • Learning Rate Decay;
  • Time Record;
  • Accuracy Visualization.

Dataset Download

We use the hot dog dataset here, which contains pictures of hot dogs and others. There are 1400 pictures in each category of the training set and 1,000 in each of the testing set. Our task is to train the network to distinguish hot dog pictures. Implementation

Since this computer does not have a GPU, it can be very time-consuming to show whole process. Hence for the sake of simplicity, the network structure which I define here is very simple, and I just run one epoch. If you have interests, try more complex CNNs such as Resnet and run more epochs to see the results.
